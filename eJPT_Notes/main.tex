\chapter*{How to Use This Document}
\addcontentsline{toc}{chapter}{How to Use This Document}

This is my personal technical journal built while working through the INE eJPT
course. Every topic follows a consistent structure designed for revision, not just
reading once.

\textbf{Each topic covers:}
\begin{itemize}
  \item What it is --- short technical definition
  \item Why it matters in pentesting
  \item Attack flow and tools
  \item Commands with inline explanations
  \item Learning output: key takeaways, enumeration mindset, common mistakes,
        real-world impact
\end{itemize}

\textbf{Callout box guide:}

\begin{whybox}[Why this matters]
Red left-border boxes explain the pentesting relevance of a tool or technique.
\end{whybox}

\begin{notebox}
Yellow left-border boxes are personal observations or things the course specifically
emphasised.
\end{notebox}

\begin{warnbox}
Red warning boxes flag things that can break systems, cause detection, or have
legal consequences if misused outside a lab.
\end{warnbox}

\begin{takebox}
Blue boxes at the end of every section contain key takeaways, the enumeration mindset
checklist, and common mistakes.
\end{takebox}
